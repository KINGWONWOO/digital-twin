> 가이드북(보강 개정판)의 내용을 그대로 유지하면서, "접속 → 생성 → 입력 → 확인" 형태의
> 실행 순서로 재구성한 문서입니다. 각 STEP 끝의 ✅ 확인 항목을 통과해야 다음 STEP으로 넘어갑니다.
>
> 기술 스택: Unreal Engine 5.6 (C++), Next.js, Supabase, Pixel Streaming 2 (WebRTC)
> 총 소요: 약 30시간 (5단계)

## PHASE 1. 백엔드 인프라 구축 — Supabase (01~04h)

### STEP 1-1. Supabase 접속 및 프로젝트 생성
1. 브라우저에서 `https://supabase.com` 접속 → 로그인(또는 GitHub 계정으로 가입).
2. 대시보드에서 **New Project** 클릭.
3. 프로젝트 이름 입력(예: `digital-twin-factory`), DB 비밀번호 설정, 리전 선택(한국이면 `Northeast Asia (Seoul)` 권장).
4. **Create new project** 클릭 후 프로비저닝 완료(1~2분) 대기.

✅ 확인: 프로젝트 대시보드(Home)가 열리고 상태가 Active로 표시되는가.

### STEP 1-2. API 키와 URL 확보
1. 좌측 메뉴 **Project Settings → API** 이동.
2. 다음 두 값을 메모장에 복사해 둔다(뒤 단계에서 계속 사용).
   - `Project URL` → `https://YOUR_PROJECT_REF.supabase.co`
   - `anon public` 키 → `eyJhbGciOi...`
3. `service_role` 키는 **절대 클라이언트 코드에 넣지 않는다**(8.1 Edge Function 전용).

✅ 확인: URL / anon 키 두 값이 확보되었는가. service_role 키는 별도 보관(미사용)인가.

### STEP 1-3. SQL 스키마 실행
1. 좌측 메뉴 **SQL Editor** 클릭 → **New query**.
2. 아래 스크립트 전체를 붙여넣고 **Run** 실행(가이드북 3.1의 완성본 — Realtime 퍼블리케이션·RLS 포함).

```sql
-- 1) 기존 객체 정리 (스크립트 재실행 안전성 확보)
DROP TABLE IF EXISTS public.factory_machines CASCADE;

-- 2) 핵심 테이블 생성
CREATE TABLE public.factory_machines (
  id           TEXT PRIMARY KEY,                          -- 엔진 매핑용 고유 ID
  machine_name TEXT NOT NULL,                             -- 디스플레이 명칭
  temperature  NUMERIC(5,1) DEFAULT 25.0 NOT NULL,        -- 센서 온도 (소수 1자리)
  status       TEXT DEFAULT 'normal' NOT NULL,            -- 설비 상태값
  updated_at   TIMESTAMPTZ DEFAULT NOW() NOT NULL,        -- 최종 변경 시각
  CONSTRAINT constraint_status_types
    CHECK (status IN ('normal', 'warning', 'error'))
);

-- 3) 상태 기반 조회 최적화 인덱스
CREATE INDEX idx_factory_machines_status
  ON public.factory_machines (status);

-- 4) updated_at 자동 갱신 트리거 (DB 레벨에서 시각 무결성 강제)
CREATE OR REPLACE FUNCTION public.set_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_set_updated_at
  BEFORE UPDATE ON public.factory_machines
  FOR EACH ROW EXECUTE FUNCTION public.set_updated_at();

-- 5) Realtime 전송 준비
ALTER TABLE public.factory_machines REPLICA IDENTITY FULL;
-- ★중요★ 이 한 줄이 없으면 구독 이벤트가 클라이언트로 전달되지 않음
ALTER PUBLICATION supabase_realtime ADD TABLE public.factory_machines;

-- 6) Row Level Security (RLS)
ALTER TABLE public.factory_machines ENABLE ROW LEVEL SECURITY;

CREATE POLICY "anon_select_machines"
  ON public.factory_machines
  FOR SELECT TO anon
  USING (true);

-- [데모 전용] 운영 전환 시 제거하고 Edge Function 경유로 대체(PHASE 5)
CREATE POLICY "anon_update_machines_demo_only"
  ON public.factory_machines
  FOR UPDATE TO anon
  USING (true)
  WITH CHECK (status IN ('normal', 'warning', 'error'));

-- 7) 시드 데이터 적재 (가상 설비 3대)
INSERT INTO public.factory_machines (id, machine_name, temperature, status)
VALUES
  ('cnc-01',      'CNC 고속 선반 A동',              42.5, 'normal'),
  ('robot-02',    '다축 다목적 조립 로봇 팔 B동',    78.2, 'warning'),
  ('conveyor-03', '메인 패키징 컨베이어 벨트',       28.0, 'normal');
```

✅ 확인 ①: **Table Editor → factory_machines** 에 3개 행(cnc-01, robot-02, conveyor-03)이 보이는가.
✅ 확인 ②: SQL Editor에서 `UPDATE public.factory_machines SET status='broken' WHERE id='cnc-01';` 실행 시 CHECK 제약으로 **거부**되는가. (거부되면 정상)
✅ 확인 ③: **Database → Publications → supabase_realtime** 에 factory_machines가 등록되어 있는가.

### STEP 2-5. MachineActor C++ 클래스 생성
1. 에디터에서 **Tools → New C++ Class → Actor** 선택, 이름 `MachineActor`로 생성.
2. `MachineActor.h`를 가이드북 4.3의 명세로 작성(색상 보간·지수 백오프 상태 포함):

```cpp
#pragma once
#include "CoreMinimal.h"
#include "GameFramework/Actor.h"
#include "Interfaces/IHttpRequest.h"
#include "MachineActor.generated.h"

UCLASS()
class DIGITALTWIN_API AMachineActor : public AActor
{
    GENERATED_BODY()
public:
    AMachineActor();
protected:
    virtual void BeginPlay() override;
    virtual void Tick(float DeltaTime) override; // 색상 보간을 위해 사용
public:
    // Supabase id 컬럼과 1:1 매핑되는 고유 기기 ID (에디터에서 개별 입력)
    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Digital Twin Config")
    FString MachineID;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Digital Twin Components")
    UStaticMeshComponent* MeshComponent;

    UPROPERTY(BlueprintReadOnly, Category = "Digital Twin Materials")
    UMaterialInstanceDynamic* DynamicMaterialInstance;

    void FetchMachineStatus();
    void OnStatusResponseReceived(FHttpRequestPtr Request,
        FHttpResponsePtr Response, bool bWasSuccessful);
private:
    FLinearColor ResolveStatusColor(const FString& Status) const;
    FTimerHandle PollingTimerHandle;
    FLinearColor TargetColor  = FLinearColor::Green;
    FLinearColor CurrentColor = FLinearColor::Green;
    float CurrentInterval = 2.0f;
    int32 ConsecutiveFailures = 0;
};
```

3. `MachineActor.cpp`를 가이드북 4.4의 완성본으로 작성. 핵심 구간:

```cpp
#include "MachineActor.h"
#include "HttpModule.h"
#include "Interfaces/IHttpResponse.h"
#include "Dom/JsonObject.h"
#include "Serialization/JsonReader.h"
#include "Serialization/JsonSerializer.h"

// ── 환경 상수: 운영에서는 .ini / 환경 변수로 분리 권장 ──
static const FString SUPABASE_URL =
    TEXT("https://YOUR_PROJECT_REF.supabase.co");
static const FString SUPABASE_ANON_KEY =
    TEXT("YOUR_SUPABASE_ANON_KEY");

AMachineActor::AMachineActor()
{
    PrimaryActorTick.bCanEverTick = true; // 색상 보간을 위해 Tick 활성화
    MeshComponent =
        CreateDefaultSubobject<UStaticMeshComponent>(TEXT("FactoryMeshComp"));
    RootComponent = MeshComponent;
}

void AMachineActor::BeginPlay()
{
    Super::BeginPlay();
    // 1) 런타임 다이내믹 머티리얼 인스턴스 생성 (예외 가드)
    if (MeshComponent && MeshComponent->GetMaterial(0))
    {
        DynamicMaterialInstance = MeshComponent->CreateDynamicMaterialInstance(0);
    }
    else
    {
        UE_LOG(LogTemp, Warning,
            TEXT("[DigitalTwin] %s: 0번 슬롯 머티리얼이 없어 색상 제어 불가."),
            *MachineID);
    }
    // 2) 폴링 타이머 구동 (2초 주기). MachineID 미설정 액터는 통신하지 않음.
    if (!MachineID.IsEmpty())
    {
        GetWorldTimerManager().SetTimer(
            PollingTimerHandle, this,
            &AMachineActor::FetchMachineStatus,
            CurrentInterval, true);
    }
}

void AMachineActor::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);
    // 상태 색상을 즉시 점프시키지 않고 부드럽게 보간
    if (DynamicMaterialInstance)
    {
        CurrentColor = FMath::CInterpTo(CurrentColor, TargetColor, DeltaTime, 6.0f);
        DynamicMaterialInstance->SetVectorParameterValue(
            TEXT("StatusColor"), CurrentColor);
    }
}

void AMachineActor::FetchMachineStatus()
{
    FHttpRequestRef Request = FHttpModule::Get().CreateRequest();
    Request->OnProcessRequestComplete().BindUObject(
        this, &AMachineActor::OnStatusResponseReceived);
    // 해당 ID 한 건만 조회 (네트워크 페이로드 최소화)
    const FString TargetUrl = FString::Printf(
        TEXT("%s/rest/v1/factory_machines?id=eq.%s&select=status,temperature"),
        *SUPABASE_URL, *MachineID);
    Request->SetURL(TargetUrl);
    Request->SetVerb(TEXT("GET"));
    Request->SetHeader(TEXT("apikey"), SUPABASE_ANON_KEY);
    Request->SetHeader(TEXT("Authorization"),
        FString::Printf(TEXT("Bearer %s"), *SUPABASE_ANON_KEY));
    Request->SetTimeout(5.0f); // 무한 대기 방지
    Request->ProcessRequest();
}

void AMachineActor::OnStatusResponseReceived(
    FHttpRequestPtr Request, FHttpResponsePtr Response, bool bWasSuccessful)
{
    // ── 네트워크 실패: 지수 백오프로 폴링 주기를 점차 늘려 서버 보호 ──
    if (!bWasSuccessful || !Response.IsValid())
    {
        ConsecutiveFailures++;
        CurrentInterval = FMath::Min(CurrentInterval * 2.0f, 30.0f);
        GetWorldTimerManager().SetTimer(PollingTimerHandle, this,
            &AMachineActor::FetchMachineStatus, CurrentInterval, true);
        UE_LOG(LogTemp, Warning,
            TEXT("[DigitalTwin] %s HTTP 실패 (%d회) → 주기 %.1fs 로 조정"),
            *MachineID, ConsecutiveFailures, CurrentInterval);
        return;
    }
    if (Response->GetResponseCode() != 200)
    {
        UE_LOG(LogTemp, Warning,
            TEXT("[DigitalTwin] 서버 에러 코드: %d"), Response->GetResponseCode());
        return;
    }
    // 통신 정상화 시 폴링 주기 원복
    if (ConsecutiveFailures > 0)
    {
        ConsecutiveFailures = 0;
        CurrentInterval = 2.0f;
        GetWorldTimerManager().SetTimer(PollingTimerHandle, this,
            &AMachineActor::FetchMachineStatus, CurrentInterval, true);
    }

    const FString Body = Response->GetContentAsString();
    TArray<TSharedPtr<FJsonValue>> JsonArray;
    TSharedRef<TJsonReader<>> Reader = TJsonReaderFactory<>::Create(Body);
    if (FJsonSerializer::Deserialize(Reader, JsonArray) && JsonArray.Num() > 0)
    {
        const TSharedPtr<FJsonObject> Obj = JsonArray[0]->AsObject();
        if (Obj.IsValid())
        {
            FString StatusValue;
            if (Obj->TryGetStringField(TEXT("status"), StatusValue))
            {
                // 즉시 적용 대신 보간 타깃만 갱신 (실제 적용은 Tick)
                TargetColor = ResolveStatusColor(StatusValue);
            }
        }
    }
}

FLinearColor AMachineActor::ResolveStatusColor(const FString& Status) const
{
    if (Status.Equals(TEXT("warning"), ESearchCase::IgnoreCase))
        return FLinearColor(1.0f, 0.4f, 0.0f, 1.0f); // 경고: 주황
    if (Status.Equals(TEXT("error"), ESearchCase::IgnoreCase))
        return FLinearColor::Red;                     // 위험/정지: 빨강
    return FLinearColor::Green;                       // 정상: 초록
}
```

### STEP 3-4. 대시보드 페이지 작성
`web/app/page.tsx` 를 가이드북 5.2 전체 소스로 교체(부분 갱신·구독 상태 표시·cleanup 포함):

```tsx
'use client';
import { useEffect, useState } from 'react';
import { supabase } from '@/lib/supabaseClient';

interface MachineData {
  id: string;
  machine_name: string;
  temperature: number;
  status: 'normal' | 'warning' | 'error';
  updated_at: string;
}

const STATUS_COLOR: Record<MachineData['status'], string> = {
  normal: '#2E8B57',
  warning: '#D2691E',
  error: '#C0392B',
};

export default function DigitalTwinControlDashboard() {
  const [machines, setMachines] = useState<MachineData[]>([]);
  const [log, setLog] = useState('시스템 초기화 중...');
  const [online, setOnline] = useState(false);

  // 초기 적재 + Realtime 구독 파이프라인 개설
  useEffect(() => {
    fetchMachines();
    const channel = supabase
      .channel('realtime-factory-layer')
      .on(
        'postgres_changes',
        { event: '*', schema: 'public', table: 'factory_machines' },
        (payload) => {
          const next = payload.new as Partial<MachineData>;
          setLog(`동기화: [${payload.eventType}] ${next?.id ?? ''}`);
          // 전체 재조회 대신 변경 행만 부분 갱신 (네트워크 절약)
          setMachines((prev) =>
            prev.map((m) => (m.id === next?.id ? { ...m, ...next } : m))
          );
        }
      )
      .subscribe((status) => {
        // 구독 상태를 UI 로 노출 → 연결 끊김을 사용자가 인지
        setOnline(status === 'SUBSCRIBED');
        if (status === 'CHANNEL_ERROR' || status === 'TIMED_OUT') {
          setLog('실시간 채널 재연결 시도 중...');
        }
      });
    // 언마운트 시 채널 해제 (메모리/소켓 누수 방지)
    return () => {
      supabase.removeChannel(channel);
    };
  }, []);

  const fetchMachines = async () => {
    const { data, error } = await supabase
      .from('factory_machines')
      .select('*')
      .order('id', { ascending: true });
    if (error) { setLog(`조회 오류: ${error.message}`); return; }
    if (data) { setMachines(data as MachineData[]); setLog('데이터 적재 완료'); }
  };

  // 가상 센서 조작 (DB 원격 갱신). 성공 시 Realtime 이 화면을 되돌려줌
  const simulate = async (
    id: string, status: MachineData['status'], temp: number
  ) => {
    const { error } = await supabase
      .from('factory_machines')
      .update({ status, temperature: temp })
      .eq('id', id);
    if (error) setLog(`제어 전송 오류: ${error.message}`);
  };

  return (
    <main style={{ maxWidth: 1100, margin: '0 auto', padding: 24,
      fontFamily: 'system-ui, sans-serif' }}>
      <h1 style={{ fontSize: 24, fontWeight: 700 }}>
        Smart Factory Digital Twin · Control Terminal
      </h1>
      <p style={{ color: '#666', fontSize: 13 }}>
        Supabase Realtime PostgreSQL Pipeline · v1.0
      </p>

      <div style={{ display: 'flex', alignItems: 'center', gap: 8,
        padding: '10px 14px', borderRadius: 8,
        background: '#0f172a', color: '#e2e8f0',
        fontFamily: 'monospace', fontSize: 13 }}>
        <span style={{ width: 9, height: 9, borderRadius: '50%',
          background: online ? '#22c55e' : '#ef4444' }} />
        SYSTEM LOG: {log}
      </div>

      <div style={{ display: 'grid',
        gridTemplateColumns: 'repeat(auto-fill, minmax(280px, 1fr))',
        gap: 16, marginTop: 20 }}>
        {machines.map((m) => (
          <div key={m.id} style={{ border: '1px solid #e5e7eb',
            borderRadius: 12, padding: 16 }}>
            <div style={{ display: 'flex', justifyContent: 'space-between' }}>
              <strong>{m.machine_name}</strong>
              <span style={{ color: STATUS_COLOR[m.status], fontWeight: 700 }}>
                {m.status.toUpperCase()}
              </span>
            </div>
            <p>센서 온도: {m.temperature} °C</p>
            <p style={{ fontSize: 12, color: '#888' }}>
              최종 동기화: {new Date(m.updated_at).toLocaleTimeString()}
            </p>
            <div style={{ display: 'flex', gap: 6, marginTop: 8 }}>
              <button onClick={() => simulate(m.id, 'normal', 26.4)}>정상</button>
              <button onClick={() => simulate(m.id, 'warning', 79.2)}>경고</button>
              <button onClick={() => simulate(m.id, 'error', 104.8)}>긴급정지</button>
            </div>
          </div>
        ))}
      </div>

      {/* UE5 픽셀 스트리밍 뷰포트 임베드 */}
      <section style={{ marginTop: 28 }}>
        <h2 style={{ fontSize: 18 }}>UE5 WebRTC Pixel Streaming Viewport</h2>
        <iframe
          src={process.env.NEXT_PUBLIC_PIXEL_STREAM_URL}
          style={{ width: '100%', height: 540, border: '1px solid #e5e7eb',
            borderRadius: 12 }}
          allow="autoplay; fullscreen"
        />
      </section>
    </main>
  );
}
```

✅ 확인 ①: `http://localhost:3000` 에서 설비 카드 3장이 표시되는가.
✅ 확인 ②: 상단 인디케이터가 **초록(SUBSCRIBED)** 인가.
✅ 확인 ③: Supabase SQL Editor에서 status를 수동 UPDATE → 새로고침 없이 카드가 즉시 갱신되는가. (안 되면 STEP 1-3의 퍼블리케이션 등록 재점검)

### STEP 3-5. 엔드투엔드(웹 → DB → 3D) 연동 검증
1. UE5 PIE 실행 상태 유지 + 대시보드 열기.
2. 대시보드의 **[경고]** / **[긴급정지]** 버튼 클릭.

✅ 확인: 버튼 클릭 → ① 대시보드 카드가 즉시 갱신(Realtime push) → ② 2초 내 UE5 메쉬 색상이 보간되며 변경(REST polling). **여기까지 되면 데모가 성립**한다 — PHASE 4가 막혀도 영상 캡처로 대체 가능(일정 운용 팁).

## PHASE 4. 픽셀 스트리밍 — WebRTC 송출 (21~26h)

> UE 5.5+ 변경 주의: 시그널링 서버는 엔진 내부에 없고 별도 저장소를 클론하며,
> `-PixelStreamingIP/-PixelStreamingPort` 는 deprecated → `-PixelStreamingURL` 사용.

### STEP 4-1. 플러그인 활성화 및 패키징
1. UE5 에디터 → **Edit → Plugins** → "Pixel Streaming" 검색 → **Pixel Streaming 2** 체크 → 에디터 재시작.
2. **Platforms → Windows → Package Project** 로 프로젝트 패키징(예: `C:\Build\`).

✅ 확인: 패키징이 완료되어 `DigitalTwinFactory.exe`(프로젝트명.exe)가 생성되었는가.

### STEP 4-2. 시그널링 서버 클론 및 구동
1. PowerShell/cmd에서 UE 버전과 **같은 브랜치**로 클론:
```bash
git clone --branch UE5.6 https://github.com/EpicGamesExt/PixelStreamingInfrastructure.git
cd PixelStreamingInfrastructure/SignallingWebServer/platform_scripts/cmd
```
2. 의존성 설치(최초 1회): `setup.bat`
3. 서버 구동: `start.bat` (기본 HTTP 80, 스트리머 WS 8888)
   - Linux/Mac은 `../bash/` 폴더의 동명 `.sh` 스크립트 사용.

✅ 확인: 콘솔에 서버 기동 로그가 뜨고 80/8888 포트가 리슨 중인가.

### STEP 4-3. 패키징 바이너리를 스트리밍 인수로 실행
`run_pixelstream.bat` 작성 후 실행:

```bat
:: PixelStreamingIP / PixelStreamingPort 는 PS2 에서 deprecated.
"C:\Build\DigitalTwinFactory.exe" ^
  -PixelStreamingURL=ws://127.0.0.1:8888 ^
  -RenderOffScreen ^
  -Unattended ^
  -AudioMixer ^
  -ForceRes -ResX=1920 -ResY=1080
:: -RenderOffScreen : 창 없이 헤드리스 렌더링 (창 최소화로 인한 스트림 중단 방지)
:: -Unattended      : 모달 다이얼로그 차단 (서버/무인 실행 안정성)
```

✅ 확인: 브라우저에서 `http://127.0.0.1` 접속 → **Play** 클릭 → 3D 화면이 스트리밍되고 마우스/키 입력이 전달되는가.
(검은 화면만 나오면: 구버전 인수 사용 여부, 인프라 브랜치-UE 버전 일치 여부 점검 — 9.2)

### STEP 4-4. (인터넷 송출 시) STUN/TURN 설정
LAN 밖으로 송출할 때만 수행. 테스트는 Google 공개 STUN으로 충분, 운영은 자체 TURN(coturn 등) 필요:

```bash
start.bat --peerConnectionOptions="{\"iceServers\":[
  {\"urls\":[\"stun:stun.l.google.com:19302\"]},
  {\"urls\":[\"turn:YOUR_TURN_HOST:3478\"],
   \"username\":\"user\",\"credential\":\"pass\"}
]}"
```

### STEP 4-5. 대시보드 iframe 결합
1. `.env.local`의 `NEXT_PUBLIC_PIXEL_STREAM_URL`이 시그널링 웹 서버 주소(`http://127.0.0.1`)인지 확인 — page.tsx의 iframe이 이를 임베드한다.

✅ 확인: 대시보드 하단 뷰포트 안에서 스트림이 재생되고, **버튼 클릭 → iframe 속 3D 색상 변화**까지 한 화면에서 확인되는가.

## PHASE 5. 최적화 · 보안 · 제출물 (27~30h)

### STEP 5-1. (확장) 중앙 Manager + 델리게이트 패턴 적용
설비 N이 커질 때 개별 폴링은 붕괴(1,000대 × 2초 = 초당 약 500커넥션)하므로, 통신 주체를 분리한다:

1. `FactoryDataManager` 액터 클래스 생성(가이드북 4.5의 .h/.cpp 그대로):
   - 레벨에 **단 1개**만 배치, 2초마다 `?select=id,status,temperature` 로 전체 설비를 Bulk 폴링.
   - 결과를 `TMap<FString, FFactoryMachineStatus>` 캐시에 적재 후 `OnFactoryDataUpdated` 멀티캐스트 델리게이트로 방송.
2. 각 `MachineActor`는 자신의 폴링 타이머를 **제거**하고, BeginPlay에서 매니저의 델리게이트에 구독만 등록 → 방송 스냅샷에서 자기 `MachineID` 상태를 꺼내 `TargetColor`만 갱신.
3. 결과: 설비 10,000개여도 네트워크 비용 **초당 0.5회 고정**.

✅ 확인: 매니저 1개 + 액터 N개 구성에서도 색상 동기화가 동일하게 작동하는가. (Output Log에 Bulk 폴링 실패 경고가 없는가)

### STEP 5-2. 운영용 보안 쓰기 경로 (Edge Function)
1. Supabase CLI로 함수 생성: `supabase functions new control-machine`
2. `supabase/functions/control-machine/index.ts` 작성(가이드북 8.1):

```ts
import { createClient } from 'jsr:@supabase/supabase-js@2';

Deno.serve(async (req) => {
  const { id, status, temperature } = await req.json();
  // 입력 검증 (화이트리스트)
  const allowed = ['normal', 'warning', 'error'];
  if (!id || !allowed.includes(status)) {
    return new Response(JSON.stringify({ error: 'invalid payload' }),
      { status: 400 });
  }
  // service_role 키는 함수 런타임 환경 변수로만 존재 (클라이언트 미노출)
  const admin = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  );
  const { error } = await admin
    .from('factory_machines')
    .update({ status, temperature })
    .eq('id', id);
  return new Response(JSON.stringify({ ok: !error, error }),
    { headers: { 'Content-Type': 'application/json' } });
});
```

3. 배포: `supabase functions deploy control-machine`
4. **운영 전환 시** 데모용 정책 제거:
```sql
DROP POLICY "anon_update_machines_demo_only" ON public.factory_machines;
```
5. 대시보드의 `simulate` 함수를 직접 UPDATE 대신 이 Edge Function 호출로 교체.

✅ 확인: anon 키 직접 UPDATE는 **거부**되고, Edge Function 경유 쓰기만 성공하는가. (읽기는 RLS, 쓰기는 서버 경유 — 이중 방어)
